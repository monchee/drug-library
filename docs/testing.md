# DREAM

**Drug Reaction Evaluation and Anaesthetic Management**

DREAM is a nurse-led digital platform designed for real-time documentation of perioperative drug allergy testing encounters at RPAH. It works alongside SCRATCH as the clinical action tool — while SCRATCH is your reference, DREAM is where you record the encounter.

[Launch DREAM :material-open-in-new:](https://dream.yuson.au){ .md-button .md-button--primary target="_blank" }

---

## How DREAM Works

Patient background information — including allergy history, prior reactions, and relevant comorbidities — is entered in REDCap first. DREAM imports this data at the start of each encounter, pre-populating the patient record so you can focus on real-time documentation during testing.

During the encounter, DREAM guides you through each step of the testing protocol and captures structured observations at each interval. On completion, it generates a clinical note ready for upload to the eMR.

```
REDCap (patient background) → DREAM (real-time encounter) → eMR (clinical note)
```

---

## Relationship to SCRATCH

SCRATCH and DREAM are complementary tools designed to be used together during allergy testing encounters.

- Use **SCRATCH** to look up drug concentrations, dilution series, and testing protocols before and during the procedure
- Use **DREAM** to document the encounter in real time as you perform testing

Neither tool stores patient-identifiable data independently. All patient data is managed within REDCap under SLHD privacy and governance frameworks.

---

## Access

DREAM is accessible from any modern browser on any ward device — no installation required. It is designed as a progressive web application (PWA) and can be added to your device's home screen for quick access.
